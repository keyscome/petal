# obsutil

## Overview

`obsutil` is Huawei Cloud's official CLI tool for interacting with Object Storage Service (OBS) — analogous to AWS's `aws s3` or Alibaba's `ossutil`. This example installs the Linux amd64 binary on all cluster nodes, making it available system-wide so that other petal tasks can fetch packages from OBS using `obsutil cp`. A separate `credential.yml` configures each node's OBS access credentials.

## Architecture

- **Topology**: 3 nodes in parallel (`node1`, `node2`, `node3`)
- **Binary path**: `/usr/bin/obsutil`
- **Config file**: `~/.obsutilconfig` (generated by `obsutil config -i <AK> -k <SK> -e <endpoint>`)
- **Download source**: Huawei Cloud OBS community CDN (`obs-community-intl.obs.ap-southeast-1.myhuaweicloud.com`)
- **Credentials**: Access Key (`OBS_AK`), Secret Key (`OBS_SK`), Endpoint (`OBS_ENDPOINT`) — supplied via environment variables or a `.env` file; **never committed to source control**

## Files

| File | Description |
|------|-------------|
| `install.yml` | Downloads `obsutil`, extracts it, and places the binary in `/usr/bin/` |
| `credential.yml` | Configures OBS credentials on all nodes using `obsutil config` |
| `uninstall.yml` | Backs up the OBS config and removes the `obsutil` binary |

### `install.yml`

1. **Download OBS Util Package** — Runs `wget` on all three nodes in parallel to download the current `obsutil_linux_amd64.tar.gz` archive from the official Huawei Cloud OBS CDN.
2. **Extract OBS Util Package** — Extracts the tarball, then moves the `obsutil` binary from the extracted directory into `/usr/bin/`, making it available on `$PATH` system-wide.
3. **Check OBS Util Cli** — Runs `which obsutil` on all three nodes to confirm the binary is accessible.

### `credential.yml`

> **Security note**: The `OBS_AK` and `OBS_SK` fields in this file are intentionally left empty. Always supply these values via a `.env` file (excluded by `.gitignore`) or the `-e` flag at runtime — never commit credentials to source control.

1. **Check Sense Env** — Echoes `$OBS_AK`, `$OBS_SK`, and `$OBS_ENDPOINT` to verify the environment variables are correctly resolved before attempting configuration.
2. **Configure OBS Credential** — Runs `obsutil config -i $OBS_AK -k $OBS_SK -e $OBS_ENDPOINT` to write the credentials to `~/.obsutilconfig` on each node.
3. **Check OBS Credential** — Prints the first 3 lines of `~/.obsutilconfig` to confirm the credentials were written correctly.

### `uninstall.yml`

1. **Ensure TMP_DIR Exists** — Creates the backup temp directory (`$TMP_DIR`) if it doesn't already exist.
2. **Move OBS Configuration File to TMP_DIR** — Moves `~/.obsutilconfig` to the backup directory to preserve credentials before removing the CLI.
3. **Check OBS Configuration File in TMP_DIR** — Confirms the config file was moved successfully.
4. **Move OBS Util CLI to TMP_DIR** — Moves `/usr/bin/obsutil` to the backup directory.
5. **Check OBS Util CLI in TMP_DIR** — Confirms the binary was moved successfully and is no longer on `$PATH`.
